Tunneling Service

An ngrok-like tunneling system running sish. You can request access by emailing tunnels@hrsn.net.

You can see the status of the tunneling service on our status page.

Contact abuse@hrsn.net for abuse of the tunneling system.

Default Subdomains

  • *.t.hrsn.net - HTTP, SSH, TCP tunnel support.
  • *.t.hrsn.dev - HTTP tunnel support.

Usage

Creating a tunnel using a default, auto-generated subdomain on t.hrsn.net which connects to http://localhost:3000

Change localhost:3000 in the below command to the connection your local web server is running on.

ssh -R 80:localhost:3000 t.hrsn.net -p 2222 -i "PATH/TO/SSH/KEY"

Choosing a custom subdomain

Creating a tunnel which uses localwebsite.t.hrsn.net which connects to http://localhost:3000

ssh -R localwebsite:80:localhost:3000 t.hrsn.net -p 2222 -i "PATH/TO/SSH/KEY"

Using a domain other than t.hrsn.net

To use another default domain such as t.hrsn.dev, append .t.hrsn.dev to the subdomain you want as shown in the command below.

ssh -R localwebsite.t.hrsn.dev:80:localhost:3000 t.hrsn.net -p 2222 -i "PATH/TO/SSH/KEY"

Advanced Usage

For more advanced usage such as TCP tunnels, SSH forwarding and more, you can read through the sish docs for more information.

If you require any help or would like a custom solution, please contact us at tunnels@hrsn.net.
